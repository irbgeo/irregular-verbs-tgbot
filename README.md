# irregular-verbs-tgbot

Telegram-бот для заучивания английских неправильных глаголов. Сначала **Тест**
по уровню быстро находит слова, которые ты ещё не знаешь, потом **Учить**
интервально тренирует их до состояния «выучено». Бот сам **напоминает**, если
ты давно не занимался.

**Бот:** [t.me/irregular_verbs_practice_bot](https://t.me/irregular_verbs_practice_bot)

---

## 🚀 Старт и онбординг

При первом `/start` бот спрашивает только **вариант форм**: британский (`gb`)
или американский (`us`) — он влияет на то, какие формы считаются верными и как
они показываются. После выбора — главное меню.

## 📋 Главное меню (5 кнопок)

1. **🧪 Тест** — выбрать уровень и пройти тест по его словам.
2. **🎓 Учить** — интервальная тренировка отобранных слов.
3. **📋 Мои слова** — слова в изучении и выученные; тап меняет статус по кругу учу→выучено→скип, скипнутые убираются из списка.
4. **📚 Список слов** — все слова по уровням; добавление/удаление из изучения.
5. **🔎 Поиск** — найти глагол по любой форме или переводу и добавить его в изучение.

## 🧪 Тест

Выбираешь уровень (Elementary … Upper-Intermediate). Тест **накопительный**:
берёт слова выбранного уровня и всех более ранних, по одному в случайном
порядке. На каждое слово бот показывает **инфинитив** (`go`), а ты
**одним сообщением вводишь все 3 формы по порядку**: base → past → participle.

Под заданием кнопки **«💡 Помощь»**, **«⏭️ Скип»**, **«⬅️ Меню»**.

- **Верно** → «✅ Верно!» с тремя формами и переводом, затем выбор
  **«✅ В изучение»** / **«⏭️ Скип»**.
- **Ошибка / «Помощь»** → бот показывает правильный ответ (3 формы + перевод);
  слово добавляется в изучение, переходим к следующему.
- **«Скип»** → слово пропускается без изменений.

Так Тест наполняет твой список изучения тем, что ты ещё не знаешь.

### Как принимается ввод
- 3 формы по порядку **base → past → participle** — **порядок важен**.
- Разделители любые (пробел, `/`, запятая, новая строка). Перед инфинитивом
  можно «to» (необязательно).
- Многовариантные формы: **обе нужны только для `was/were`** (past у *be*).
  Прочие альтернативы (hung/hanged, burnt/burned …) — достаточно любой одной.
- Регистр и лишние пробелы не важны; опечатки не прощаются.

Примеры: `go went gone` ✓ · `to go went gone` ✓ · `went go gone` ✗ (порядок) ·
`be was were been` ✓ · `burn burnt burnt` ✓.

## 🎓 Учить

Тренируем слова из изучения (а также иногда — уже выученные, для повторения).

### Один круг = один под-вопрос
Показывается **якорь** — одна случайная форма слова (base, past или participle),
и просят **одну** другую форму. Перевод в заданиях не участвует.

- Инфинитив показывается без «to» (`🎓 go`), варианты формы — через «/» (`was/were`).
- У **выученного** слова на повторении вместо 🎓 показывается **🔁**.
- Подпись с названием формы у якоря не показывается — только значение.

После ответа бот показывает **«✅ Верно!»** / **«❌ Неверно»** / **«💡»** с тремя
формами и переводом. Контролы: **«💡 Показать»** и **«⬅️ Меню»**. «Скипа» нет.

### Два режима (ступени)
Формат ответа зависит от ступени слова:

- **Режим 1 — выбор кнопками.** Цель выбираешь из **4 вариантов** (1 верный +
  3 неверных из частых ошибок слова и форм других глаголов).
- **Режим 2 и повторение — свободный ввод.** Цель печатаешь текстом (правила —
  как в Тесте: «to» у инфинитива; обе формы только для `was/were`).

### Коробки Лейтнера и переходы
У слова есть «коробка» 0–5 в текущем режиме:

- ответ верный → коробка **+1**;
- ошибка или **«Показать»** → коробка **0**, бот показывает верные формы,
  переходим к следующему слову;
- коробка дошла до **5 в режиме 1** → слово переходит в **режим 2** (коробка 0);
- коробка дошла до **5 в режиме 2** → слово **выучено**;
- выученное слово на повторении: верно → без изменений; **ошибка** → возвращается
  в изучение (режим 2, коробка 0).

> Слово, добавленное в изучение через списки, начинает «Учить» с **режима 1**.

### Выбор слова и сессия
- Сессия **бесконечная** — слово за словом, пока не нажмёшь **«⬅️ Меню»**.
- Следующее слово — взвешенно: **90%** из изучаемых, **10%** из выученных
  (повторение). Одно и то же слово не повторяется чаще, чем раз в **5** слов.
- Если учить нечего (нет ни изучаемых, ни выученных) — экран **«Пока нечего
  учить»** с кнопками «🧪 Тест» и «⬅️ Меню».

## 🔔 Напоминания

Если у тебя есть слова для учёбы и ты **больше 24 часов не решал задание**, бот
сам присылает готовое задание «Учить» — отвечаешь прямо в чате. Пока молчишь —
напоминание приходит не чаще **раза в сутки**; как только снова порешал, отсчёт
начинается заново. Новым пользователям даётся сутки форы.

## 🏷️ Статусы слова

| Статус | Значение |
|--------|----------|
| новое | ещё не встречалось (нет в твоём списке) |
| в изучении | тренируется в «Учить»: режим 1 или 2, коробка 0–5 |
| выучено | прошёл режим 2 (коробка 5 в режиме 2) |
| скип | отмечено «не учить» |

## 📊 Полный цикл слова

1. **Тест**: ошибка / «Помощь» → слово в **изучение** (режим 1). При верном
   ответе сам выбираешь «в изучение» или «скип». Либо добавляешь слово вручную
   через **«Список слов»**.
2. **Учить, режим 1** (выбор) → коробка 5 → **режим 2**.
3. **Учить, режим 2** (ввод) → коробка 5 → **выучено**.
4. Выученное иногда подкидывается на повторение; ошибка возвращает в изучение
   (режим 2, коробка 0).
5. Простаиваешь > 24ч при наличии слов → **напоминание** с заданием.

## ⚙️ Запуск

Бот работает на long polling; данные — в MongoDB; каталог глаголов грузится из
`data/verbs.json`.

Скопируй `.env.example` в `.env` и заполни: `BOT_TOKEN`, `MONGO_USERNAME`,
`MONGO_PASSWORD`. `.env` в git не коммитится.

```bash
cp .env.example .env   # затем впиши токен и креды Mongo
docker compose up -d --build
```

**MongoDB с паролем.** Mongo поднимается с аутентификацией (root-пользователь из
`MONGO_USERNAME`/`MONGO_PASSWORD`), порт наружу **не публикуется** — база
доступна только боту внутри compose-сети. Бот подключается строкой
`mongodb://USER:PASS@mongo:27017/?authSource=admin`.

> Аутентификация включается только на **чистом** томе: при первом запуске с auth
> на уже существующих данных выполни `docker compose down -v && docker compose up -d`.

Конфигурация — через переменные окружения (`envconfig`, см. `internal/config`):
`BOT_TOKEN` (обязателен), `MONGO_URI`, `MONGO_DB`, `VERBS_PATH`.

**Store-тесты** ходят в Mongo по `MONGO_URI` (по умолчанию `localhost:27017`).
Так как порт не публикуется, с хоста они **скипаются**; чтобы прогнать — задай
`MONGO_URI` с кредами на доступный Mongo (или временно опубликуй порт).

Документация по этапам — в `docs/superpowers/specs/`.
