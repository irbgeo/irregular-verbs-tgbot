# Списки: убрать перевод из подписей + инфо о слове по тапу

Дата: 2026-06-30

## Цель

Доработать оба экрана списков — «Мои слова» (`KindMyWords`) и «Список слов»
(`KindWordList`):

1. В подписях слов **не показывать перевод**.
2. По тапу на слово, помимо текущего переключения статуса, **показывать инфо о
   слове** (все 3 формы + перевод) в тексте сообщения.

## Поведение

### Подписи слов
Кнопка слова: `<иконка> <base> - <past> - <participle>`.
Раньше в конце был ` - <translation>` — убираем. Касается обоих экранов.

### Тап по слову (вариант A)
`tog:<base>` работает как сейчас (переключает статус в черновике), и
дополнительно под заголовком сообщения показывает блок инфо о нажатом слове:

```
📋 Мои слова

be - was/were - been
быть, являться, находиться
```

- Формы — по варианту пользователя (gb/us), как в списке; альтернативы через `/`.
- Перевод — список через запятую.
- Инфо показывается **только сразу после тапа**. При листании страниц (`lp:`),
  смене секции (`sec:`), открытии списка — заголовок без блока инфо («гаснет»).
- Неизвестная база — как сейчас: ре-рендер без изменений, без блока инфо.
- Тап может перенести слово в другую секцию/страницу — блок инфо всё равно
  показывает нажатое слово (инфо не зависит от страницы).

## Реализация

- `ListView` (internal/service/types.go): добавить поле `Selected *ListItem`.
- `ListToggle` (internal/service/lists.go): после построения view проставить
  `v.List.Selected` для нажатой базы (формы — через существующий `itemForms`).
  Остальные list-операции `Selected` не ставят (nil → нет блока).
- `screens.go`: `wordRows` без перевода; `renderMyWords` и `renderWordList`
  дорисовывают блок инфо, если `l.Selected != nil`.

## Тесты (TDD)

- service: `ListToggle` возвращает view, у которого `List.Selected` — нажатое
  слово с правильными формами/переводом по варианту; другие операции
  (`ListPage`, `ListSection`) оставляют `Selected == nil`.
- bot: подписи слов не содержат перевод; при `Selected != nil` рендер добавляет
  блок инфо (формы + перевод) в текст. Обновить существующие тесты подписей.

## Вне scope

Формат/набор форм не меняем (3 формы), новых данных в `verbs.json` не добавляем.
