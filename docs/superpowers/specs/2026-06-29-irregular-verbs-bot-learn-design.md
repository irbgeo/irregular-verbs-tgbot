# Технический дизайн: «Учить» (этап 3)

Дата: 2026-06-29. Уточняет/заменяет раздел «Учить» в
`2026-06-28-irregular-verbs-bot-v2-design.md` (§6.1–6.2, §8).

## 1. Цель и место в проекте

«Учить» — интервальная тренировка слов, отобранных в Тесте (`study`) и уже
выученных (`learned`, повторение). Два режима с коробками Лейтнера. Это
последний крупный этап MVP (Тест → Учить).

Главное отличие от Теста: круг = **один** под-вопрос; очередь не конечна —
слова выбираются по весам, сессия идёт, пока пользователь не выйдет в меню.

## 2. Вход и сессия

Тап **🎓 Учить** (`menu:learn`):

- Пул для тренировки = слова со статусом `study` (любой `mode`) **∪** `learned`.
- Пул пуст → экран `learn_empty` («Пока нечего учить 🙂») с кнопками
  «🧪 Тест» (`menu:test`) и «⬅️ Меню» (`nav:menu`).
- Пул не пуст → сразу старт: выбрать слово (`pickLearnWord`), собрать круг
  (якорь+цель), показать вопрос (экран `quiz`).

Сессия **бесконечная**: после каждого слова берём следующее. Выход —
кнопка «⬅️ Меню» (`nav:menu`, очищает `State`). Пул не может опустеть по ходу
сессии: повышение по лестнице оставляет слово в пуле (`study`→`study` или
`study`→`learned`, оба входят в пул), поэтому экран «всё выучено» недостижим,
пока есть хотя бы одно слово — отдельного сообщения для него не делаем.

## 3. Выбор слова (`pickLearnWord`)

- Группы: `S` = слова `study`, `L` = слова `learned`.
- Бросок веса: `rng(100) < 90` → берём из `S`, иначе из `L`.
  - Если выбранная группа пуста — берём из другой.
- Антиповтор: в `Session.Recent` хранится кольцо последних **5** показанных
  слов (база). Из группы выбираем случайное слово, **не входящее** в кольцо.
  Если все кандидаты группы в кольце — кольцо игнорируем (берём любое).
- Случайность — через `s.rng` (детерминируется в тестах).

После выбора слова кольцо обновляется: текущая база добавляется в `Recent`,
хранятся последние 5 (старые вытесняются).

## 4. Круг (один под-вопрос)

### Якорь и цель

- **Якорь** — случайно один из 4 видов: `base`, `past`, `participle`,
  `translation`. Показываем его значение с подписью.
- **Цель** — случайный вид из пула:
  - `{base, past, participle}` **всегда**;
  - **плюс** `translation`, только если якорь ≠ `translation`.
- Якорь-форма входит в пул цели → цель может совпасть с якорем (тогда ответ
  виден на экране — это допустимо). Якорь-`translation` → `translation` из
  пула цели исключён.
- Итог: якорь-форма → 4 кандидата цели; якорь-`translation` → 3 кандидата.

Отображение значений:
- `base` → `v.Base`;
- `past` → `strings.Join(v.Past[variant], "/")`;
- `participle` → `strings.Join(v.Participle[variant], "/")`;
- `translation` → `strings.Join(v.Translations, ", ")`.

### Результат круга

- **Верно** (один ответ совпал с целью) → круг пройден → переход по лестнице
  (см. §5), затем следующее слово.
- **Неверно** или нажата **«💡 Показать»** → круг оборван: `box=0` по правилам
  лестницы, показываем верные формы+перевод (`correctText`), затем следующее
  слово. Других под-вопросов нет (он один).

### Формат ответа (выводится из прогресса слова)

| Прогресс слова | Формат | Режим |
|---|---|---|
| `study`, `mode 1` | `choice` (кнопки) | режим 1 |
| `study`, `mode 2` | `input` (текст) | режим 2 |
| `learned` | `input` (текст) | повторение |

**Режим 2 / повторение (`input`):** цель вводится текстом. Приём (нормализация
`lower+trim`, опечатки не прощаются):
- `base` → `input == v.Base`;
- `past` → любая из `v.Past[variant]`;
- `participle` → любая из `v.Participle[variant]`;
- `translation` → любой из `v.Translations`.

**Режим 1 (`choice`):** цель выбирается тапом по кнопке. Тот же круг из одного
под-вопроса, только варианты:
- цель-форма (`base`/`past`/`participle`): **4 кнопки** = 1 верная +
  3 неверных. Неверные: сначала из `common_mistakes` слова, добор — случайные
  формы (того же вида, если возможно) других глаголов. Все различны, порядок
  случайный.
- цель-`translation`: **5 кнопок** = 1 верный перевод + 4 случайных перевода
  других слов. Все различны, порядок случайный.
- Верное значение кнопки-формы — `v.Past[variant][0]` / `v.Participle[variant][0]`
  / `v.Base`; верный перевод — `v.Translations[0]`. Проверка: значение нажатой
  кнопки равно верному.

## 5. Лестница Лейтнера

`BoxMax = 5`. Применяется к `WordProgress` текущего слова по итогу круга.

- **`study`, `mode 1`:**
  - верно → `box++`; если `box == 5` → `mode = 2, box = 0`;
  - ошибка → `box = 0`.
- **`study`, `mode 2`:**
  - верно → `box++`; если `box == 5` → `status = learned, mode = 0, box = 0`;
  - ошибка → `box = 0`.
- **`learned` (повторение):**
  - верно → без изменений;
  - ошибка → `status = study, mode = 2, box = 0`.

Лестница в целом: `study(mode1)` →(5 верных)→ `study(mode2)` →(5 верных)→
`learned`. Любая ошибка обнуляет коробку текущего режима; ошибка на повторении
возвращает слово в `study(mode2)`.

## 6. Экраны, контролы, callback'и

| Экран | Содержимое |
|---|---|
| `quiz` (учить) | Текст: якорь (вид+значение) и вопрос про цель. Режим 1 — ряды кнопок-вариантов. Контролы: `[💡 Показать]=quiz:help` · `[⬅️ Меню]=nav:menu`. (В «Учить» **нет** «Скип».) |
| `learn_empty` | «Пока нечего учить 🙂» · `[🧪 Тест]=menu:test` · `[⬅️ Меню]=nav:menu` |

- Режим 1: каждый вариант — кнопка с новым префиксом `lc:<idx>` (индекс в
  показанном списке вариантов; значение сверяется в сервисе).
- Режим 2/повторение: ответ — текстовое сообщение.
- `quiz:help` в «Учить» = «Показать» (раскрыть формы) = ошибка круга.
- Меню остаётся 4 кнопки; `menu:learn` больше не заглушка.

## 7. Изменения модели

`internal/service/types.go`:

- `Session` — поля для «учить» (с `bson:",omitempty"`):
  - `AnchorKind string` — вид якоря (`base`/`past`/`participle`/`translation`);
  - `TargetKind string` — вид цели;
  - `Options []string` — варианты режима 1 (порядок как на экране), пусто для
    `input`;
  - `Recent []string` — кольцо кулдауна (последние 5 баз).
  - Поля `Level`/`Queue`/`Step` для «учить» не используются (`Step` остаётся
    тест-полем).
- `QuizView` — поля для рендера «учить»:
  - `Mode string` (`test`/`learn`), `Format string` (`input`/`choice`);
  - `AnchorKind`, `AnchorValue string`; `TargetKind string`;
  - `Options []string` (режим 1).
- `WordProgress{Status,Mode,Box}` и `BoxMax` — **без изменений**.

## 8. Сервис (use-cases) и проверки

`internal/service` (без импорта адаптеров; единственный писатель `User`):

- `StartLearn(ctx, userID)` — пул пуст → `View{Screen: learn_empty}`; иначе
  `pickLearnWord`, собрать круг, `View{Screen: quiz, Quiz: learnQuestion(...)}`.
- `LearnAnswer(ctx, userID, text)` — только если `inLearn` и `Format==input`:
  проверить цель → лестница(success/fail) → `advanceLearn`; при ошибке
  `Feedback` с `correctText`.
- `LearnChoose(ctx, userID, idx)` — только если `inLearn` и `Format==choice`:
  значение `Options[idx]` против верного → лестница → `advanceLearn`.
- `LearnReveal(ctx, userID)` — «Показать»: лестница(fail) + `Feedback` +
  `advanceLearn`.
- `advanceLearn(u)` — обновить `Recent`, `pickLearnWord`, собрать новый круг;
  вернуть `View{Screen: quiz, ...}`.
- `pickLearnWord(u) (base string, ok bool)` — выбор по §3.
- `buildRound(u, base)` — выбрать `AnchorKind`/`TargetKind` по §4, для `choice`
  собрать `Options`.
- `checkTarget(v, kind, input, variant) bool` — проверка по §4 (формы/перевод).
  Отдельная от тестовой `checkAnswer` (та после правки Теста перевод не
  проверяет).
- `learnLadder(u, base, ok bool)` — переходы §5.
- Хелперы дистракторов: `formOptions`, `translationOptions` (используют `rng`,
  `common_mistakes`, каталог).

`internal/bot`:

- роутер: `menu:learn` → `StartLearn`; `lc:<idx>` → `LearnChoose`; текст при
  `learn`-сессии → `LearnAnswer`; `quiz:help` при `learn` → `LearnReveal`.
- рендер `quiz` для `learn`: заголовок якоря, вопрос цели, кнопки вариантов
  (режим 1), контролы `[💡 Показать][⬅️ Меню]`; рендер `learn_empty`.

## 9. Тестирование (TDD)

- `pickLearnWord`: веса 90/10 (детерминированный `rng`), пустая группа →
  другая, кулдаун (исключение кольца; все в кольце → игнор).
- `buildRound`: пул цели (якорь-форма → 4; якорь-перевод → 3),
  `translation` исключён при якоре-переводе.
- `checkTarget`: формы (любая валидная форма варианта, base), перевод (любой
  из translations), нормализация, опечатки.
- `formOptions`/`translationOptions`: размер (4/5), верный включён, все
  различны.
- `learnLadder`: все переходы (mode1 box++; mode1 box5→mode2; mode2 box5→
  learned; ошибка→box0; learned верно→без изменений; learned ошибка→study/
  mode2/box0).
- Интеграция роутера: `menu:learn` → quiz; верный ответ (input и choice) →
  следующее слово + лестница; «Показать» → ошибка; `nav:menu` → меню, сессия
  очищена; пустой пул → `learn_empty`.

## 10. Разбивка на под-этапы (для плана)

- **A.** Модель (`Session`/`QuizView`), `pickLearnWord`, `buildRound`,
  `checkTarget`, `learnLadder`, `StartLearn`/`LearnAnswer`/`LearnReveal`/
  `advanceLearn`, роутинг текста и `menu:learn`, рендер `quiz`(input) и
  `learn_empty`. Режим 2 и повторение работают сквозняком.
- **B.** Режим 1: `formOptions`/`translationOptions`, `Options` в круге,
  `LearnChoose`, рендер кнопок, `lc:<idx>` в роутере.
- **C.** Шлифовка: ровный текст заданий/фидбэка, проверка краёв (пул из 1
  слова, кулдаун при малом пуле), ручной smoke.

## 11. Вне объёма

- Изменения `data/verbs.json` (дубликаты/странные записи) — отдельно.
- Статистика/прогресс-бары, напоминания, смена настроек — не в этом этапе.
- Тест и списки — не трогаем (кроме общих хелперов рендера, если переиспользуем).
