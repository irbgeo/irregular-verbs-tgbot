## 1. Цель

Переосмысление концепции. Две активности:

- **Тест** по уровню — слова уровня в случайном порядке, круг из 4 заданий со
  свободным вводом; «Помощь»/«Скип»; ошибка или помощь → слово в изучение;
  всё верно → спросить «в изучение / скип».
- **Учить** — интервальная тренировка отобранных слов по коробкам Лейтнера,
  взвешенный выбор (90% изучаемые / 10% выученные), антиповтор (кулдаун 5).

## 2. Архитектура (переиспользуем Этап 1)

Ports & adapters без изменений: `internal/service` (вся логика), адаптеры
`internal/store` (Mongo), `internal/bot` (Telegram), `internal/config`,
композиция `cmd/bot`. `service` не импортирует адаптеры; в `users` пишет только
`service`. Каталог слов — в памяти (из `LoadVerbs`).

**Новое для бота:** обработка **текстовых сообщений** как ответов в квизе (не
только команд и callback'ов). Роутер по `state.screen` решает, что текст — это
ответ на текущее задание.

## 3. Статусы слова и коробки

Статус хранится в карте `words` пользователя; слова «новое» в карте нет.

| Статус | Коробка | Значение |
|--------|---------|----------|
| (нет записи) | — | новое, ещё не встречалось |
| `study` | 0–4 | в изучении |
| `learned` | 5 | выучено |
| `skipped` | — | «не учить» |

Константы: `StatusStudy`, `StatusLearned`, `StatusSkipped`. Коробки: `0..5`,
`BoxLearned = 5`.

## 4. Модель данных

```jsonc
{
  "_id": 123456789,
  "settings": { "variant": "gb" },          // только вариант форм (gb|us)
  "state": {
    "screen": "quiz",
    "session": {                              // активная сессия квиза (тест или учить)
      "mode": "test",                         // "test" | "learn"
      "level": "elementary",                  // для теста: уровень
      "queue": ["take","go", ...],            // для теста: оставшиеся слова (random-порядок)
      "base": "take",                         // текущее слово
      "step": 1,                              // индекс под-вопроса 0..3
      "used_help": false,                     // в текущем круге нажимали «Помощь»
      "recent": ["be","do","get","make","see"]// последние слова (для кулдауна, режим learn)
    }
  },
  "words": {
    "take": { "status": "study",   "box": 2 },
    "be":   { "status": "learned", "box": 5 },
    "do":   { "status": "skipped", "box": 0 }
  },
  "created_at": "...", "last_active_at": "..."
}
```

- `session` присутствует только во время квиза; вне квиза пуст.
- `recent` — кольцо последних показанных слов длиной 5 (для антиповтора в «Учить»).
- Поля `mode1_streak`/`current_round` из старого дизайна не используются.

## 5. Каталог и порядок

`service.New(users UserRepository, verbs []Verb)` строит индекс: `byBase`,
`byLevel` (для теста). Тестовая очередь — слова уровня в **случайном** порядке;
перестановка засевается из `userID`+номера запуска (детерминируемо для тестов
через инъекцию источника случайности; см. §11).

## 6. Квиз-движок (общий для теста и учёбы)

Круг по слову = 4 под-вопроса (`step` 0..3):

| step | Показываем | Просим ввести | Принимаем |
|------|-----------|---------------|-----------|
| 0 | перевод (рус) | инфинитив (base) | `base` |
| 1 | `BASE` | past | любую форму `past[variant]` |
| 2 | `BASE` | past participle | любую форму `participle[variant]` |
| 3 | `BASE` | перевод | любой из `translations` |

**Проверка ответа** (`CheckAnswer`): нормализация `strings.ToLower(strings.TrimSpace(x))`;
сравнение с нормализованными допустимыми значениями; опечатки не прощаются.

Круг «верный», только если **все 4** под-вопроса верны. Первая ошибка
завершает круг (показываем правильные формы и перевод).

## 7. Поток «Тест»

1. `OpenTest` → экран выбора уровня.
2. `StartTest(level)` → строим `session{mode:test, level, queue:shuffle(words(level)), base:queue.pop, step:0, used_help:false}`; экран `quiz`.
3. Текстовый ответ (`Answer(text)`):
   - неверно → показать правильное; слово → `study/box0`; завершить круг; следующее слово (`advance`).
   - верно и `step<3` → `step++`, следующий под-вопрос.
   - верно и `step==3`:
     - если `used_help` → слово → `study/box0`; `advance`.
     - иначе → экран `test_result` с вопросом «В изучение / Скип».
4. `Help` → показать 3 формы + перевод; `used_help=true`; слово → `study/box0`;
   `advance` (к следующему слову).
5. `Skip` → слово не меняем (если не было записи — остаётся «новое»); `advance`.
6. `test_result`: `Keep` → `study/box0`; `Drop` → `skipped`. Затем `advance`.
7. `advance`: если очередь пуста → экран `test_done` (итог + «Меню»); иначе
   следующее слово, `step=0`, `used_help=false`.

## 8. Поток «Учить»

Право на тренировку: слова `study` (box 0–4) и `learned` (box 5).

1. `OpenLearn`:
   - нет ни одного `study`-слова → экран `learn_empty` («Все слова выучены»
     или «Пока нечего учить») с кнопкой **«Пройти тест»** (→ выбор уровня) и
     «Меню».
   - иначе выбрать слово (`pickLearnWord`) и открыть `quiz` (`mode:learn`).
2. `pickLearnWord` (взвешенный выбор с кулдауном):
   - кандидаты = слова `study` ∪ `learned`, **исключая** входящие в `recent`
     (последние 5); если после фильтра пусто — игнорируем кулдаун.
   - бросок: с вероятностью 90% берём из `study`, 10% — из `learned`
     (если одной группы нет — из доступной); внутри группы — равновероятно.
3. Ответы — тот же круг из 4 (`Answer`). Конец круга:
   - все верно → `box = min(box+1, 5)`; `box==5` → `status=learned`.
   - ошибка → `box=0`, `status=study` (выученное возвращается в изучение).
   - В «Учить» доступна **«💡 Показать»** (раскрыть формы) — считается как
     ошибка круга (box→0). Кнопки «Скип»/«в изучение» в учёбе нет.
4. После круга: добавить слово в `recent` (хвост 5), выбрать следующее
   (`pickLearnWord`) или, если `study` опустел, → `learn_empty`.
5. Кнопка «Меню» завершает сессию (`session` очищается).

## 9. Экраны и callback/ввод

Экраны (`Screen`): `onboarding_variant`, `main_menu`, `test_level`, `quiz`,
`test_result`, `test_done`, `learn_empty`, `my_words`.

| Экран | Содержимое |
|-------|-----------|
| `onboarding_variant` | «Выберите вариант форм:» · `[🇬🇧 British]=variant:gb` `[🇺🇸 American]=variant:us` |
| `main_menu` | «Главное меню:» · `[🧪 Тест]=menu:test` · `[🎓 Учить]=menu:learn` · `[📋 Мои слова]=menu:mywords` |
| `test_level` | «Выберите уровень:» · кнопка на каждый уровень `[<Label>]=level:<lvl>` · `[⬅️ Меню]=nav:menu` |
| `quiz` | Текст задания (см. §6) + подсказка «введите ответ». Кнопки: тест — `[💡 Помощь]=quiz:help` `[⏭️ Скип]=quiz:skip` `[⬅️ Меню]=nav:menu`; учёба — `[💡 Показать]=quiz:help` `[⬅️ Меню]=nav:menu`. **Ответ — текстовое сообщение.** |
| `test_result` | «Верно! Добавить в изучение?» · `[✅ В изучение]=res:keep` `[⏭️ Скип]=res:drop` |
| `test_done` | «Тест уровня пройден.» · `[⬅️ Меню]=nav:menu` |
| `learn_empty` | «Все слова выучены 🎉» / «Пока нечего учить.» · `[🧪 Пройти тест]=menu:test` `[⬅️ Меню]=nav:menu` |
| `my_words` | Заголовок + фильтр `[В изучении]=mw:study` `[Выучено]=mw:learned` `[Скип]=mw:skipped`; список `• BASE` (для study — «(коробка N)»); `[⬅️ Меню]=nav:menu` |

**Маршрутизация ввода (роутер):**
- `Message.Text == "/start"` → `Start` (онбординг или меню).
- Прочий `Message.Text`, когда `state.screen == "quiz"` → `Answer(text)`.
- `CallbackQuery` → по `kind:value`: `variant→SetVariant`, `menu:test→OpenTest`,
  `menu:learn→OpenLearn`, `menu:mywords→OpenMyWords`, `level:<lvl>→StartTest`,
  `quiz:help→Help`, `quiz:skip→Skip`, `res:keep→Keep`, `res:drop→Drop`,
  `mw:<filter>→MyWordsFilter`, `nav:menu→OpenMenu`.

Текст вне квиза игнорируется (тихо). Невалидный callback/ввод → тихо
подтвердить, экран не ломается.

## 10. Онбординг (изменение Этапа 1)

Онбординг сокращается до одного шага — выбор варианта (`gb`/`us`). Шаги
«уровень» и «порядок» из Этапа 1 убираются. Завершение → главное меню (3 кнопки).

## 11. Тестирование (TDD)

`service` (фейковый `UserRepository`, реальный каталог, инъекция источника
случайности `rand` для детерминизма):
- проверка ответа: формы варианта, любой перевод, нормализация, опечатки — мимо;
- тест: построение очереди (случайность детерминирована); круг из 4; ошибка/
  помощь → `study/box0`; всё верно → `test_result`; `keep/drop`; конец очереди →
  `test_done`;
- учить: `pickLearnWord` — веса 90/10, кулдаун 5 (не повторяет из `recent`),
  деградация при пустой группе; верный круг → box+1, box5→`learned`; ошибка →
  box0/`study`; нет `study` → `learn_empty`;
- `my_words` фильтры.

`bot`: `render` экранов; маршрутизация текстового ответа в `quiz`; кнопки.

## 12. Definition of Done

- Unit-тесты проходят; `go vet` чист.
- Ручной тест: онбординг (вариант) → Тест уровня (ввод ответов, Помощь/Скип,
  «в изучение/скип») → Учить (коробки растут, выученное подкидывается, кулдаун) →
  «все выучены» предлагает тест → «Мои слова» с фильтрами.

## 13. Этапы (заново выведены) — см. дорожную карту

- **Этап 1 (готов):** фундамент (бот, Mongo, ports&adapters).
- **Этап 2:** онбординг-вариант, меню (3 кнопки), модель статусов/коробок,
  «Мои слова» с фильтрами, экран выбора уровня (без квиза).
- **Этап 3:** квиз-движок (круг из 4, свободный ввод, проверка) + **Тест**
  (Помощь/Скип, наполнение изучения).
- **Этап 4:** **Учить** — коробки Лейтнера, взвешенный выбор 90/10, кулдаун 5,
  «все выучены → тест».
- **Этап 5:** напоминания (24 ч) и т. п.

## 14. Ключевые решения (на проверку)

- Единица «верно/неверно» — целый круг (4 под-вопроса); первая ошибка завершает
  круг.
- Коробки 0..5, 5 = выучено; ошибка всегда → 0; выученное при ошибке → в изучение.
- В тесте: помощь/ошибка → `study/box0`; всё верно без помощи → спросить
  «в изучение/скип»; «скип» при «новом» оставляет слово новым (не пишем `skipped`?)
  — **решение:** «Скип» в `test_result` пишет `skipped`; кнопка «⏭️ Скип» в самом
  задании (до ответа) просто пропускает слово без записи.
- Под-вопрос 0: перевод→инфинитив; 1–3: base→past/participle/перевод.
- «Учить»: есть «Показать» (= ошибка круга), нет «Скип»/«в изучение».
- Антиповтор — кольцо `recent` длиной 5; выбор группы 90/10, внутри — равновероятно.
- Старый «Список слов» как отдельный экран не делаем; всё через «Мои слова» с
  фильтрами.
