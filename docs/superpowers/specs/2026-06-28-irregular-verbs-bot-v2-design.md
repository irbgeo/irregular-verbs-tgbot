## 1. Цель

Две активности:

- **Тест** по уровню — слова уровня в случайном порядке; круг из 4 заданий со
  свободным вводом; «Помощь»/«Скип»; ошибка или помощь → слово в изучение;
  всё верно → спросить «в изучение / скип».
- **Учить** — интервальная тренировка отобранных слов, **два режима** с
  коробками Лейтнера; взвешенный выбор (90% изучаемые / 10% выученные),
  антиповтор (кулдаун 5).

## 2. Архитектура (переиспользуем Этап 1)

Ports & adapters без изменений: `internal/service` (вся логика), адаптеры
`internal/store` (Mongo), `internal/bot` (Telegram), `internal/config`,
композиция `cmd/bot`. `service` не импортирует адаптеры; в `users` пишет только
`service`. Каталог слов — в памяти (из `LoadVerbs`).

**Новое для бота:** обработка **текстовых сообщений** как ответов в режиме
свободного ввода (роутер по `state.screen`).

## 3. Статусы, режимы, коробки

`words[base]` = `{ status, mode, box }`.

| Статус | Значение |
|--------|----------|
| (нет записи) | новое, ещё не встречалось |
| `study` | в изучении: режим `mode` (1 или 2), коробка `box` 0–5 |
| `learned` | выучено (прошёл режим 2) |
| `skipped` | «не учить» |

- `mode` ∈ {1, 2} — текущий режим изучения (только при `study`).
- `box` ∈ 0..5 — коробка Лейтнера в текущем режиме.
- **Лестница:** `study(mode1)` → коробка 5 → `study(mode2, box0)` → коробка 5 →
  `learned`. Ошибка в любом режиме → `box=0`. Выученное при ошибке (на
  повторении) → `study(mode2, box0)`.

Константы: `StatusStudy`/`StatusLearned`/`StatusSkipped`; `BoxMax = 5`.

## 4. Модель данных

```jsonc
{
  "settings": { "variant": "gb" },           // только вариант форм
  "state": {
    "screen": "quiz",
    "session": {
      "mode": "test",                          // "test" | "learn"
      "format": "input",                       // "input" (тест, учить-режим2) | "choice" (учить-режим1)
      "level": "elementary",                   // тест: уровень
      "queue": ["take","go", ...],             // тест: оставшиеся слова (random)
      "base": "take",                          // текущее слово
      "step": 1,                               // под-вопрос 0..3
      "anchor": "past",                        // учить: показанная «якорная» форма
      "used_help": false,                      // тест: была ли помощь в круге
      "options": ["took","take","taken","made"],// choice: варианты текущего под-вопроса
      "recent": ["be","do","get","make","see"] // учить: кольцо последних 5 (кулдаун)
    }
  },
  "words": {
    "take": { "status": "study",   "mode": 1, "box": 2 },
    "be":   { "status": "study",   "mode": 2, "box": 4 },
    "go":   { "status": "learned", "mode": 0, "box": 0 },
    "do":   { "status": "skipped", "mode": 0, "box": 0 }
  }
}
```

`session` существует только во время квиза.

## 5. Каталог и порядок

`service.New(users, verbs []Verb)` строит индексы: `byBase`, `byLevel`. Тестовая
очередь — слова уровня в случайном порядке (источник случайности инъектируется
для детерминизма тестов). Дистракторы (режим 1) берутся из каталога.

## 6. Квиз-движок: два формата

Круг по слову = 4 под-вопроса. Проверка форм — по варианту (`gb`/`us`);
проверка перевода — по списку `translations`. Круг «верный» только если **все
4** под-вопроса верны; первая ошибка завершает круг (показываем правильные
формы и перевод).

### 6.1 Свободный ввод (`format=input`) — Тест и Учить-режим 2

Ответ — текстовое сообщение; нормализация `lower+trim`; опечатки не прощаются.

- **Тест:** под-вопросы фиксированы: (0) перевод→инфинитив; (1) base→past;
  (2) base→participle; (3) base→перевод.
- **Учить-режим 2:** показывается случайная якорная форма (`anchor`), нужно
  ввести все 4 формы слова (base, past, participle, перевод) по очереди.

Принимаем: для форм — любую валидную форму `past[variant]`/`participle[variant]`
(а также сам `base`); для перевода — любой из `translations`.

### 6.2 Множественный выбор (`format=choice`) — Учить-режим 1

Показывается случайная якорная форма; **поочерёдно** выбрать base, past,
participle, перевод — каждый из вариантов-кнопок.

- **Дистракторы форм:** 4 варианта = 1 верный + 3 неверных из `common_mistakes`
  слова и форм других глаголов (добор из каталога), в случайном порядке.
- **Дистракторы перевода:** 5 вариантов = верный перевод + 4 случайных перевода
  других слов.
- Проверка — по нажатой кнопке (callback несёт индекс/значение).

## 7. Поток «Тест»

1. `OpenTest` → экран выбора уровня.
2. `StartTest(level)` → `session{mode:test, format:input, level, queue:shuffle(level), base, step:0, used_help:false}`; экран `quiz`.
3. Ответ (текст): неверно → показать верное, слово → `study/mode1/box0`, `advance`;
   верно и `step<3` → `step++`; верно и `step==3` → если `used_help` →
   `study/mode1/box0` + `advance`, иначе экран `test_result` («в изучение / скип»).
4. `Help` → показать формы+перевод; `used_help=true`; слово → `study/mode1/box0`; `advance`.
5. `Skip` (кнопка в задании) → слово не трогаем; `advance`.
6. `test_result`: `Keep` → `study/mode1/box0`; `Drop` → `skipped`; затем `advance`.
7. `advance`: очередь пуста → `test_done`; иначе следующее слово, `step=0`, `used_help=false`.

## 8. Поток «Учить»

Право на тренировку: `study` (mode1/mode2) ∪ `learned`.

1. `OpenLearn`: нет ни одного `study`-слова → экран `learn_empty` («Все слова
   выучены 🎉» / «Пока нечего учить») с кнопками «🧪 Пройти тест» и «Меню».
   Иначе выбрать слово (`pickLearnWord`) и открыть `quiz`.
2. `pickLearnWord` (взвешенно + кулдаун): кандидаты = `study` ∪ `learned`, минус
   входящие в `recent` (последние 5; если после фильтра пусто — кулдаун
   игнорируем). Бросок: 90% из `study`, 10% из `learned` (если группы нет —
   из доступной); внутри группы равновероятно. `format` = по слову: `study`
   mode1 → `choice`, mode2 → `input`; `learned` → `input` (повторение).
3. Круг (`Answer`/выбор):
   - **mode1 (`choice`)**: круг верный → `box+1`; `box==5` → `mode=2, box=0`;
     ошибка → `box=0`.
   - **mode2 (`input`)**: круг верный → `box+1`; `box==5` → `status=learned`;
     ошибка → `box=0`.
   - **learned (повторение, `input`)**: верно → без изменений; ошибка →
     `status=study, mode=2, box=0`.
   - В «Учить» есть «💡 Показать» (раскрыть формы) = ошибка круга. «Скип»/
     «в изучение» в учёбе нет.
4. После круга: добавить слово в `recent` (хвост 5); если `study` опустел →
   `learn_empty`; иначе следующее слово.
5. «Меню» завершает сессию (`session` очищается).

## 9. Экраны и маршрутизация

Экраны: `onboarding_variant`, `main_menu`, `test_level`, `quiz`, `test_result`,
`test_done`, `learn_empty`, `my_words`.

| Экран | Содержимое |
|-------|-----------|
| `onboarding_variant` | «Выберите вариант форм:» · `[🇬🇧 British]=variant:gb` `[🇺🇸 American]=variant:us` |
| `main_menu` | `[🧪 Тест]=menu:test` · `[🎓 Учить]=menu:learn` · `[📋 Мои слова]=menu:mywords` |
| `test_level` | кнопка на уровень `[<Label>]=level:<lvl>` · `[⬅️ Меню]=nav:menu` |
| `quiz` (input) | Текст задания. Кнопки: тест — `[💡 Помощь]=quiz:help` `[⏭️ Скип]=quiz:skip` `[⬅️ Меню]=nav:menu`; учить — `[💡 Показать]=quiz:help` `[⬅️ Меню]=nav:menu`. Ответ — текст. |
| `quiz` (choice) | Текст задания + кнопки-варианты `[<вариант>]=opt:<i>` · `[💡 Показать]=quiz:help` `[⬅️ Меню]=nav:menu` |
| `test_result` | «Верно! Добавить в изучение?» · `[✅ В изучение]=res:keep` `[⏭️ Скип]=res:drop` |
| `test_done` | «Тест уровня пройден.» · `[⬅️ Меню]=nav:menu` |
| `learn_empty` | «Все слова выучены 🎉» / «Пока нечего учить.» · `[🧪 Пройти тест]=menu:test` `[⬅️ Меню]=nav:menu` |
| `my_words` | Заголовок + фильтр `[В изучении]=mw:study` `[Выучено]=mw:learned` `[Скип]=mw:skipped`; список `• BASE` (study — «(режим M, коробка N)»); `[⬅️ Меню]=nav:menu` |

**Роутер:** `/start`→`Start`; прочий текст при `screen==quiz`→`Answer(text)`;
callbacks: `variant→SetVariant`, `menu:test→OpenTest`, `menu:learn→OpenLearn`,
`menu:mywords→OpenMyWords`, `level→StartTest`, `quiz:help→Help`,
`quiz:skip→Skip`, `opt:<i>→Choose`, `res:keep→Keep`, `res:drop→Drop`,
`mw:<f>→MyWordsFilter`, `nav:menu→OpenMenu`. Текст вне квиза и невалидные
callback'и — тихо подтвердить.

## 10. Онбординг (изменение Этапа 1)

Сокращается до одного шага — выбор варианта (`gb`/`us`). Шаги «уровень»/
«порядок» убираются. Завершение → главное меню (3 кнопки).

## 11. Тестирование (TDD)

`service` (фейк `UserRepository`, реальный каталог, инъекция `rand`):
- проверка ответов (формы/перевод/нормализация/опечатки);
- генерация дистракторов (достаточность, наличие верного, нет дублей);
- тест: очередь (детерминированно), круг, ошибка/помощь → `study/mode1/box0`,
  всё верно → `test_result`, `keep/drop`, конец → `test_done`;
- учить: `pickLearnWord` (веса 90/10, кулдаун 5, деградация); mode1 box5→mode2;
  mode2 box5→learned; ошибка→box0; learned-ошибка→`study/mode2/box0`; нет study→
  `learn_empty`;
- `my_words` фильтры.

`bot`: `render` экранов; маршрутизация текстового ответа; кнопки-варианты.

## 12. Definition of Done

Unit-тесты + `go vet` чисты. Ручной тест: онбординг(вариант) → Тест уровня
(ввод, Помощь/Скип, в изучение/скип) → Учить (режим1 выбор → коробка 5 →
режим2 ввод → коробка 5 → выучено; повторение выученных; кулдаун) → «все
выучены»→тест → «Мои слова» с фильтрами.

## 13. Этапы (крупно: Тест → Учить) — см. дорожную карту

- **Этап 1 (готов):** фундамент.
- **Этап 2 — Тест (целиком):** онбординг-вариант, меню (3), модель
  `{status,mode,box}`, выбор уровня, квиз-движок свободного ввода + проверка,
  поток теста (Помощь/Скип/в изучение/скип), наполнение `study`.
- **Этап 3 — Учить (целиком):** режим 1 (выбор + дистракторы), режим 2 (ввод),
  коробки Лейтнера по режимам, лестница mode1→mode2→learned, выбор 90/10 +
  кулдаун 5, повторение выученных, «все выучены → тест».
- **Этап 4 — Мои слова + напоминания:** экран «Мои слова» с фильтрами;
  напоминания (24 ч) — `internal/reminder` + `Notifier`.

## 14. Ключевые решения (на проверку)

- Единица «верно/неверно» — целый круг (4 под-вопроса); первая ошибка завершает.
- Лестница изучения: режим1 (выбор) → коробка 5 → режим2 (ввод) → коробка 5 →
  выучено. Ошибка всегда → коробка 0 текущего режима.
- Повторение выученных идёт форматом свободного ввода (режим 2); ошибка
  возвращает в `study/mode2/box0`.
- Режим 1 (выбор): якорь — случайная форма; выбираем все 4 (base/past/participle/
  перевод). Формы — 4 варианта (1 верный + 3 дистрактора из common_mistakes/
  других слов); перевод — 5 вариантов (1 верный + 4 чужих перевода).
- Режим 2 / Тест (ввод): тест — фиксированные под-вопросы; режим 2 — якорь
  случайный, ввод всех 4 форм.
- Антиповтор — кольцо `recent` длиной 5; выбор группы 90/10, внутри равновероятно.
- Отдельного «Список слов» нет; всё через «Мои слова» с фильтрами.
- В тесте «⏭️ Скип» (в задании) пропускает без записи; «Скип» в `test_result`
  пишет `skipped`.
