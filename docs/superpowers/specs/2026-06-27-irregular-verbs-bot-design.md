## 1. Цель

Telegram-бот для заучивания английских неправильных глаголов. Пользователь
выбирает уровень, отбирает слова, проходит два режима тренировки (множественный
выбор и свободный ввод) до состояния «выучено». Бот напоминает о занятиях и
предлагает переходить на следующий уровень.

## 2. Технологический стек

- **Язык:** Go.
- **Telegram:** библиотека `github.com/irbgeo/go-tgbot` (свой zero-dependency
  клиент), режим **long polling** — один процесс с циклом `GetUpdates`.
  Публичный URL/TLS не нужен.
- **БД:** MongoDB, драйвер `go.mongodb.org/mongo-driver`.
- **Конфиг через env:** `BOT_TOKEN`, `MONGO_URI`, `MONGO_DB`.

## 3. Архитектура (пакеты)

| Пакет | Ответственность |
|-------|-----------------|
| `cmd/bot` | Точка входа: конфиг, подключение к Mongo, сидинг слов, запуск polling и шедулера. |
| `internal/store` | Доступ к Mongo: репозитории `users` и `verbs`. |
| `internal/verbs` | Сидинг из `data/verbs.json`, доступ к словам, генерация дистракторов. |
| `internal/learn` | Логика обучения: серии, проверка ответов Режима-1 и Режима-2, переходы статусов. Не зависит от Telegram. |
| `internal/bot` | Роутер апдейтов, FSM состояний пользователя, рендер экранов и клавиатур. |
| `internal/reminder` | Фоновый шедулер напоминаний (24 ч бездействия). |

**Принцип:** бизнес-логика (`learn`, `verbs`) отделена от Telegram I/O, чтобы её
можно было покрыть unit-тестами без сети.

## 4. Модель данных (MongoDB)

### Коллекция `verbs`

Сид из `data/verbs.json` при старте (upsert по `_id`). `_id = base`.

```jsonc
{
  "_id": "take",
  "base": "take",
  "level": "elementary",
  "past":       { "gb": ["took"],  "us": ["took"] },
  "participle": { "gb": ["taken"], "us": ["taken"] },
  "translations": ["брать", "взять"],
  "common_mistakes": ["make", "give"]
}
```

### Коллекция `users`

`_id = telegram user id`. Прогресс хранится **вложенной картой `words`** внутри
документа пользователя (для 170 слов это компактно и проще, чем отдельная
коллекция; один документ — один пользователь).

```jsonc
{
  "_id": 123456789,
  "settings": {
    "level":   "elementary",          // выбранный уровень
    "variant": "gb",                  // "gb" | "us"
    "order":   "random"               // "alpha" | "random"
  },
  "state": {                          // FSM: текущий экран/шаг диалога
    "screen":  "mode1",
    "payload": { /* контекст текущего вопроса */ }
  },
  "words": {
    "take": {
      "status": "mode1",              // browsing|want|mode1|mode2|learned|skipped
      "mode1_streak": 2,              // успешных кругов подряд (0..5)
      "mode2_streak": 0,
      "current_round": {              // незаконченный круг Режима-1/2
        "remaining_questions": ["past", "participle", "translation"],
        "answered_ok": true
      }
    }
  },
  "last_active_at":   "2026-06-27T10:00:00Z",
  "last_reminder_at": "2026-06-26T09:00:00Z",
  "created_at":       "2026-06-20T08:00:00Z"
}
```

**Состояние диалога (FSM) хранится в Mongo** → переживает перезапуск бота.

**Статусы слова:**

| Статус | Значение |
|--------|----------|
| `browsing` | показано в Учебном материале, решение не принято |
| `want` | отобрано к изучению, ждёт Режим-1 |
| `mode1` | проходит Режим-1 (множественный выбор) |
| `mode2` | проходит Режим-2 (свободный ввод) |
| `learned` | выучено ✅ |
| `skipped` | «Не хочу учить» |

## 5. Экраны и FSM

Главное меню — 4 кнопки: **Учебный материал · Начать обучение · Мои слова ·
Список слов**.

Экраны FSM:

```
onboarding_level → onboarding_variant → onboarding_order → main_menu
main_menu → material | mode1/mode2 | my_words | word_list
```

Каждый callback/текст роутится по полю `state.screen`. После обработки экран
обновляется в Mongo.

### 5.1 Первый запуск (`/start`)

1. Выбор уровня (elementary / pre-int / int / upper-int / advanced /
   proficiency) — это начальный пул слов.
2. Выбор варианта форм (британский `gb` / американский `us`).
3. Выбор порядка изучения (по алфавиту / рандомно).
4. Переход в главное меню.

## 6. Логика обучения

### 6.1 Учебный материал

- Показывает по одному слову уровня со статусом `browsing`:
  `🔤 BASE (past, participle)` + перевод + кнопки `✅ Хочу учить` / `❌ Не хочу`.
- «Хочу учить» → статус `want` (попадёт в Режим-1).
- «Не хочу» → статус `skipped` (исчезает из этого режима, остаётся в Списке слов).
- После каждых **5 добавленных** слов → предложение «Начать обучение?».
  Пользователь продолжает просмотр или переходит к обучению.

### 6.2 Режим-1 (множественный выбор)

- **Круг** = 4 вопроса по одному слову в порядке:
  базовая форма → past → participle → перевод.
- Бот показывает контекст и 4 кнопки (1 правильная + 3 неправильных).
- **Дистракторы:** неправильные варианты берутся из смеси `common_mistakes`
  слова и форм других глаголов того же уровня, в **случайном порядке** кнопок.
  Если `common_mistakes` не хватает — добор формами других глаголов.
- **Зачёт круга:** все 4 ответа верны → `mode1_streak += 1`. Любая ошибка →
  круг провален, `mode1_streak = 0`, незаконченный круг сбрасывается.
- **Чередование:** после завершения круга по слову X берётся другое активное
  слово (не дрилим X подряд). Внутри одного круга вопросы идут по слову X.
- **Переход:** при `mode1_streak == 5` слово переходит в `mode2` и больше не
  появляется в Режиме-1.

### 6.3 Режим-2 (свободный ввод)

- Тот же круг из 4 вопросов, но пользователь пишет ответ текстом.
- **Проверка только точная** после нормализации (lowercase + trim):
  - **Форма** — совпадение с любой валидной формой выбранного варианта (gb/us).
    Для форм с несколькими вариантами (напр. `was`/`were`) засчитывается любая.
  - **Перевод** — совпадение с любым элементом `translations`.
  - Опечатки **не** допускаются.
- Логика серий и кругов та же: 5 успешных кругов подряд → `status = learned`,
  слово отмечается как выучено и больше в обучении не появляется.

### 6.4 Выбранный вариант (gb/us)

`settings.variant` определяет, какие формы показываются как правильные и
принимаются в Режиме-2.

## 7. Напоминания (24 ч)

- Фоновый шедулер (`internal/reminder`): тикер раз в N минут сканирует
  пользователей, у кого `now - last_active_at ≥ 24 ч` и `last_reminder_at`
  достаточно старый.
- Шлёт одно случайное слово из текущих (`mode1`/`mode2`) + кнопку
  «Начать обучение». Обновляет `last_reminder_at`.

## 8. Переход на следующий уровень

- Когда доля `learned` слов текущего уровня достигает порога (по умолчанию
  **80 %**), бот предлагает добавить слова следующего уровня.
- Согласие → переход в Учебный материал нового уровня.

## 9. Список слов

- Экран показывает все слова базы, сгруппированные по уровням, с метками:
  `учу` / `пропустил` / `выучено ✅`.
- Постранично (170 слов не помещаются в одно сообщение).

## 10. Обработка ошибок

- **Telegram API:** ошибки логируются; при `retry-after` бот ждёт указанное
  время и повторяет.
- **Mongo:** ошибки логируются; пользователю показывается дружелюбное сообщение,
  состояние не теряется (FSM в БД).

## 11. Тестирование (TDD)

Тесты пишутся до реализации:

- `internal/learn`: серии (накопление и сброс), проверка ответов Режима-1 и
  Режима-2, нормализация ввода, переходы статусов (`want → mode1 → mode2 →
  learned`).
- `internal/verbs`: сидинг, генерация дистракторов (достаточность, отсутствие
  дублей, наличие правильного варианта).
- Telegram I/O мокается; сетевые вызовы в unit-тестах не выполняются.

## 12. Объём MVP

**Входит в первую версию:**

- Онбординг (уровень / вариант / порядок).
- Учебный материал, Режим-1, Режим-2 до статуса «выучено».
- Список слов по уровням с метками.
- Дневные напоминания (24 ч).
- Предложение следующего уровня.

**Вне MVP (отложено):**

- Экран «Мои слова» с детальным прогрессом (Режим + счёт X/5). На первом этапе
  кнопка может вести к простому списку текущих слов без подробного прогресса.
- Масштабирование на несколько инстансов (пока один процесс, long polling).

## 13. Принятые допущения

- Прогресс хранится картой `words` внутри документа `users` (не отдельной
  коллекцией).
- Один инстанс бота (long polling).
- Внутри круга Режима-1/2 вопросы идут по одному слову подряд (4 шт.);
  чередуются круги между разными словами.
