# Технический дизайн: Списки слов («Мои слова» + «Список слов по уровням»)

- **Дата:** 2026-06-28
- **Статус:** на согласовании
- **Расширяет:** Этап 4 дорожной карты
  ([2026-06-27-irregular-verbs-bot-roadmap.md](2026-06-27-irregular-verbs-bot-roadmap.md))
- **Базовый дизайн:** [2026-06-28-irregular-verbs-bot-v2-design.md](2026-06-28-irregular-verbs-bot-v2-design.md)
- **Продукт:** [README.md](../../../README.md)

## 1. Цель

Два экрана управления словами с интерактивным редактированием (черновик +
подтверждение):

1. **«Мои слова»** — два раздела: **«Изучаю»** (`study` + `learned`) и
   **«Скипнутые»** (`skipped`). Тап по слову переносит его в другой раздел.
2. **«Список слов»** — все 170 слов по уровням; тап помечает слово «в изучение»
   (toggle). Видны текущие статусы.

Оба экрана: тапы копятся в **черновике**, применяются по **✅ Подтвердить**,
сбрасываются по **❌ Отмена**.

## 2. Меню (4 кнопки)

`main_menu`: `[🧪 Тест]=menu:test` · `[🎓 Учить]=menu:learn` ·
`[📋 Мои слова]=menu:mywords` · `[📚 Список слов]=menu:list`.

(Кнопки «Мои слова» и «Список слов» сейчас заглушки — этот дизайн их реализует;
«Учить» остаётся заглушкой до Этапа 3.)

## 3. Статусы и иконки

`new` (нет записи) ▫️ · `study` 📘 · `learned` ✅ · `skipped` ❌.

«Изучаю» = слова со статусом `study` **или** `learned`. «Скипнутые» = `skipped`.

## 4. Модель данных (черновик в состоянии)

В `State` добавляется состояние редактирования списка (живёт только на этих
экранах):

```go
type State struct {
    Screen  string
    Session *Session   // квиз (Тест/Учить)
    List    *ListState // редактирование списков
}

type ListState struct {
    Kind    string            // "my_words" | "word_list"
    Section string            // my_words: "study" | "skipped" (активный раздел)
    Page    int               // текущая страница (0-based)
    Draft   map[string]string // base -> целевой статус ("study"|"skipped"|"new")
}
```

- В `Draft` лежат только изменённые слова (base → целевой статус).
- **Эффективный статус** слова = `Draft[base]` если есть, иначе сохранённый
  `words[base].Status` (или `new`, если записи нет).
- В `words` пишет только `service`; до «Подтвердить» туда ничего не уходит.

## 5. Контракт `View` (добавление)

```go
type View struct {
    // ... существующее (Screen, Quiz, Levels, Notice, Feedback) ...
    List *ListView
}

type ListView struct {
    Kind         string     // "my_words" | "word_list"
    Section      string     // my_words: активный раздел
    StudyCount   int        // my_words: для подписи переключателя
    SkippedCount int
    Title        string     // word_list: «Уровень — стр. i/N»
    Page, Pages  int
    HasPrev      bool
    HasNext      bool
    Items        []ListItem
}

type ListItem struct {
    Base   string
    Status string // эффективный статус (бот выбирает иконку)
}
```

`bot.render` строит экран из `ListView`: по кнопке на слово (`tog:<base>`),
ряд навигации и ряд Подтвердить/Отмена.

## 6. Экран «Мои слова» (`my_words`)

Переключатель раздела + до 10 слов активного раздела + навигация + действия.

```
📋 Мои слова
[• Изучаю (7) ]  [ Скипнутые (3) ]
📘 TAKE
✅ GO
📘 GET
[◀]  [▶]
[✅ Подтвердить]  [❌ Отмена]
```

- Переключатель: `[Изучаю (N)]=sec:study` `[Скипнутые (M)]=sec:skipped` (точка/
  маркер на активном). Смена раздела сбрасывает страницу в 0, черновик хранит.
  N/M считаются по **эффективному** статусу.
- Список активного раздела: слова с эффективным статусом раздела,
  **отсортированы по алфавиту (по `base`)**, 10/стр., кнопка
  `[<иконка> BASE]=tog:<base>`.
- **Тап (`tog`)** переносит слово в другой раздел (в черновике):
  - из «Изучаю» (`study`/`learned`) → `Draft[base]="skipped"`;
  - из «Скипнутые» → `Draft[base]="study"` (**только статус**; `mode`/`box` не
    трогаем).
  - Если тап возвращает слово к его исходному сохранённому статусу — запись из
    `Draft` удаляется (нет ложного изменения).
- Навигация: `[◀]=lp:<page-1>` `[▶]=lp:<page+1>` (только если есть страница).
- Действия: `[✅ Подтвердить]=list:ok` `[❌ Отмена]=list:cancel`.
- Пустой раздел → «Пусто».

## 7. Экран «Список слов» (`word_list`)

Все 170 слов в порядке уровней, 10/страница; заголовок — уровень страницы.

```
📚 Список слов — Elementary (стр. 1/17)
📘 BE        ← помечено/уже в изучении
▫️ BECOME
✅ BEGIN     ← выучено
❌ BREAK     ← скип
[◀]  [▶]
[✅ Подтвердить]  [❌ Отмена]
```

- 10 слов/стр. в порядке `Levels`, **внутри уровня — по алфавиту (по `base`)**.
  Заголовок = уровень первого слова страницы + «стр. i/N» (N = ceil(170/10) = 17).
- Кнопка `[<иконка эффективного статуса> BASE]=tog:<base>`.
- **Тап (`tog`)** — toggle «в изучение» (в черновике):
  - если эффективный статус ≠ `study` → `Draft[base]="study"` (только статус);
  - если эффективный статус = `study` → снять: вернуть к исходному —
    `"skipped"` если сохранённый был `skipped`, иначе `"new"`. (Если это
    возвращает к сохранённому статусу — удалить из `Draft`.)
- Навигация и действия — как в §6.

## 8. Применение черновика

- **`list:ok` (`CommitList`)**: для каждого `base→target` в `Draft`:
  - `target == "study"` → `words[base].Status = "study"` (сохранить `mode`/`box`;
    если записи не было — создать `{Status:study, Mode:0, Box:0}`);
  - `target == "skipped"` → `words[base] = {Status:skipped}`;
  - `target == "new"` → удалить `words[base]`.
  Затем очистить `State.List`, экран → `main_menu`.
- **`list:cancel` (`CancelList`)**: очистить `State.List`, экран → `main_menu`.

> Примечание: слово, добавленное в `study` со `mode=0`, корректно стартует в
> «Учить» (Этап 3) с режима 1 — там `mode==0` трактуется как начало. В списках
> `mode`/`box` не задаются (правка пользователя: «только study»).

## 9. Use-cases (service) — все возвращают `(View, error)`

| Метод | Логика |
|-------|--------|
| `OpenMyWords(userID)` | `State.List = {Kind:my_words, Section:study, Page:0, Draft:{}}`; вернуть `my_words`. |
| `OpenWordList(userID)` | `State.List = {Kind:word_list, Page:0, Draft:{}}`; вернуть `word_list`. |
| `ListSection(userID, section)` | (my_words) сменить раздел (`study`/`skipped`), `Page=0`. |
| `ListPage(userID, page)` | сменить страницу (в границах). |
| `ListToggle(userID, base)` | обновить `Draft` по правилам §6/§7 для текущего `Kind`. |
| `CommitList(userID)` | применить `Draft` к `words`, очистить `List`, `main_menu`. |
| `CancelList(userID)` | очистить `List`, `main_menu`. |

Все читают слова из in-memory каталога; список «Мои слова» строится из
`words` пользователя, «Список слов» — из всего каталога.

## 10. Экраны и callback-данные

Экраны: добавляются `my_words`, `word_list`.

Карта `kind` → use-case (в дополнение к существующим):
`menu:mywords→OpenMyWords`, `menu:list→OpenWordList`,
`sec:<study|skipped>→ListSection`, `lp:<n>→ListPage`,
`tog:<base>→ListToggle`, `list:ok→CommitList`, `list:cancel→CancelList`.

Все действия на этих экранах — через `Edit` сообщения (callback), как в Тесте.
Невалидный `base`/страница/секция → тихо подтвердить, экран не ломается.

## 11. Обработка ошибок

- Тап по слову не из текущего набора, выход страницы за границы, неизвестная
  секция → тихо подтвердить callback; черновик не портится.
- Ошибки Mongo логируются; черновик хранится в `State` (в БД) → переживает
  перезапуск.

## 12. Тестирование (TDD)

`service` (фейковый `UserRepository` + реальный каталог):
- `OpenMyWords`/`OpenWordList` инициализируют `List` (черновик пуст, стр. 0);
- `my_words`: разделение по эффективному статусу; `learned` в «Изучаю»;
  счётчики разделов; `ListSection` сбрасывает страницу;
- `ListToggle` (my_words): study/learned→skipped, skipped→study (только
  статус), возврат к исходному удаляет запись из `Draft`;
- `ListToggle` (word_list): не-study→study, study→исходный (skipped/new);
- `ListPage` границы; пагинация по 10;
- `CommitList`: применяет study (сохраняя mode/box; создаёт при отсутствии),
  skipped, new (удаление); очищает черновик; `CancelList` отбрасывает;
- ничего не пишется в `words` до `CommitList`.

`bot`: `render` экранов `my_words`/`word_list` (иконки статусов, переключатель
разделов со счётчиками, пагинация, кнопки Подтвердить/Отмена); маршрутизация
новых callback'ов.

## 13. Definition of Done

- Unit-тесты проходят; `go vet` чист.
- Ручной тест: меню (4 кнопки) → «Мои слова» (переключение Изучаю/Скипнутые,
  перенос тапом, Подтвердить/Отмена) → «Список слов» (по уровням, toggle в
  изучение с пометками, Подтвердить/Отмена). Изменения видны после повторного
  открытия.

## 14. Ключевые решения

- Перенос/пометка в «изучаю» ставит **только** `status=study`; `mode`/`box`
  не трогаем (ими управляет «Учить»).
- В «Мои слова» «Изучаю» = `study`+`learned`; тап по `learned` → `skipped`.
- «Список слов»: снятие из study → `skipped` (если был скип) иначе `new`.
- Пагинация по 10 на обоих экранах; «Список слов» — сквозь уровни в порядке
  `Levels`, заголовок = уровень страницы.
- Слова во всех списках отсортированы **по алфавиту (по `base`)**: в «Мои
  слова» — оба раздела; в «Список слов» — внутри каждого уровня.
- Редактирование — черновик в `State.List`, применяется по «Подтвердить».
- Отдельного выбора уровня в «Список слов» нет (просто листание по 10).

## 15. Вне объёма

- Логика «Учить» (Этап 3) и напоминания (Этап 4) — отдельно.
- Изменение `mode`/`box` из списков — нет (только статус).
