# Технический дизайн: Этап 1 (запуск, онбординг, экран «Мои слова»)

- **Дата:** 2026-06-27
- **Статус:** на согласовании
- **Базовый дизайн:** [2026-06-27-irregular-verbs-bot-design.md](2026-06-27-irregular-verbs-bot-design.md)
- **Источник требований:** [README.md](../../../README.md)

## 1. Цель этапа

Тонкий вертикальный срез MVP. После этого этапа бот:

1. **Запускается** — читает конфиг, подключается к MongoDB, сидит слова из
   `data/verbs.json`, крутит цикл long polling (`GetUpdates`).
2. **Онбордит пользователя** — по `/start` пользователь выбирает уровень,
   вариант форм (GB/US) и порядок изучения; профиль сохраняется в Mongo.
3. **Показывает экран «Мои слова»** — пока без обучения, поэтому экран в
   состоянии «пусто» (заглушка с текстом).

Этот этап — подмножество утверждённого дизайна (Подход A). Логика обучения,
напоминания и отбор слов в этот этап **не входят** и будут добавлены позже без
переделки уже написанного.

## 2. Объём этапа

**Входит:**

- Точка входа и запуск процесса (конфиг, Mongo, сидинг, polling, graceful
  shutdown).
- Онбординг: `onboarding_level → onboarding_variant → onboarding_order →
  main_menu`.
- Главное меню (минимальное): одна кнопка **«📋 Мои слова»**.
- Экран «Мои слова» в состоянии «пусто» + кнопка возврата в меню.
- `docker-compose.yml` для локальной MongoDB.
- Unit-тесты (TDD) на сидинг и FSM онбординга.

**Не входит (следующие этапы):**

- Режим-1, Режим-2, статусы слов, отбор слов («Учебный материал»).
- Экран «Список слов» по уровням.
- Напоминания (24 ч), переход на следующий уровень.
- Пакеты `internal/learn` и `internal/reminder`.

## 3. Архитектура (пакеты этапа)

| Пакет | Ответственность |
|-------|-----------------|
| `cmd/bot` | Точка входа: env-конфиг, подключение к Mongo, сидинг слов, цикл `GetUpdates` с offset, graceful shutdown по SIGINT/SIGTERM. |
| `internal/store` | Доступ к Mongo: `VerbRepo` (сидинг/upsert) и `UserRepo` (чтение/сохранение профиля и состояния FSM). |
| `internal/verbs` | Загрузка и парсинг `data/verbs.json`, тип `Verb`, порядок уровней, `Seed(ctx, repo)`. |
| `internal/bot` | Роутер апдейтов, FSM онбординга, рендер экранов и inline-клавиатур, экран «Мои слова». Интерфейс `Sender` оборачивает Telegram-клиент для мокирования. |

**Принцип:** бизнес-логика (переходы FSM) не зависит от сети — Telegram I/O
скрыт за интерфейсом `Sender`, поэтому покрывается unit-тестами без сети.

## 4. Модель данных (подмножество этапа)

### Коллекция `verbs`

Сид из `data/verbs.json` при старте, upsert по `_id = base`. Все 170 слов, 6
уровней. Структура документа — как в базовом дизайне.

### Коллекция `users`

`_id = telegram user id`. На этом этапе храним только профиль и состояние FSM.
Карта `words` ещё **не** добавляется (появится вместе с обучением).

```jsonc
{
  "_id": 123456789,
  "settings": {
    "level":   "elementary",   // выбранный уровень
    "variant": "gb",           // "gb" | "us"
    "order":   "alpha"         // "alpha" | "random"
  },
  "state": {
    "screen": "main_menu"      // текущий экран FSM
  },
  "created_at":     "2026-06-27T10:00:00Z",
  "last_active_at": "2026-06-27T10:00:00Z"
}
```

Состояние FSM хранится в Mongo → переживает перезапуск бота.

## 5. Онбординг и экраны (FSM)

Поток экранов:

```
onboarding_level → onboarding_variant → onboarding_order → main_menu → my_words
```

Каждый callback роутится по полю `state.screen`; после обработки экран
обновляется в Mongo, сообщение редактируется (`EditMessageText`).

1. **`/start` (новый пользователь)** → создаём документ, `screen =
   onboarding_level`, показываем 6 кнопок уровней
   (elementary / pre-intermediate / intermediate / upper-intermediate /
   advanced / proficiency).
2. **Выбор уровня** → сохраняем `settings.level`, `screen =
   onboarding_variant`, показываем кнопки **GB / US**.
3. **Выбор варианта** → сохраняем `settings.variant`, `screen =
   onboarding_order`, показываем кнопки **«По алфавиту» / «Случайно»**.
4. **Выбор порядка** → сохраняем `settings.order`, `screen = main_menu`,
   показываем главное меню.
5. **`/start` (уже онбордился)** → просто показываем главное меню; профиль не
   сбрасывается. Сброс профиля можно добавить позже.

**Главное меню (минимум этапа):** одна кнопка **«📋 Мои слова»**.

**Экран «Мои слова»:** текст-заглушка
*«У вас пока нет слов в изучении. Скоро здесь появятся слова.»* +
кнопка **«⬅️ Меню»** для возврата.

> Допущение: меню на этом этапе содержит одну кнопку («Мои слова»). Полное меню
> из 4 кнопок появится, когда будут готовы соответствующие экраны.

## 6. Конфиг и запуск

- **Env:** `BOT_TOKEN` (обязателен), `MONGO_URI` (по умолчанию
  `mongodb://localhost:27017`), `MONGO_DB` (по умолчанию `irregular_verbs`).
- **`docker-compose.yml`:** сервис `mongo` + volume для локальной разработки.
- **Модуль:** `github.com/irbgeo/irregular-verbs-tgbot`; зависимости —
  `github.com/irbgeo/go-tgbot`, `go.mongodb.org/mongo-driver`.
- **Запуск:** `docker compose up -d` → задать `BOT_TOKEN` →
  `go run ./cmd/bot`.

## 7. Обработка ошибок

- **Telegram API:** ошибки логируются; при `retry_after` бот ждёт указанное
  время и повторяет; цикл `GetUpdates` при ошибке делает backoff и продолжает.
- **Mongo:** ошибки логируются; пользователю показывается дружелюбное
  сообщение; состояние FSM не теряется (хранится в БД).

## 8. Тестирование (TDD — тесты до кода)

- **`internal/verbs`:** парсинг всех 170 слов, корректные уровни,
  идемпотентный upsert (повторный сидинг не плодит дублей).
- **`internal/bot`:** переходы онбординга (каждый шаг сохраняет нужную
  настройку), идемпотентность `/start` для онбордившегося пользователя, рендер
  пустого экрана «Мои слова». `Sender` мокается, `UserRepo` — in-memory фейк.
- **`internal/store`:** тонкий интеграционный тест с Mongo, пропускается, если
  Mongo недоступна.

## 9. Definition of Done

- Unit-тесты проходят.
- `docker compose up -d` поднимает Mongo.
- Ручной smoke-тест в Telegram: `/start` → уровень → вариант → порядок → меню →
  «Мои слова» показывает пустое состояние.

## 10. Принятые допущения

- Главное меню этапа — одна кнопка «Мои слова» (полное меню позже).
- Модуль: `github.com/irbgeo/irregular-verbs-tgbot`.
- Один инстанс бота (long polling), как в базовом дизайне.
- Карта `words` в документе `users` на этом этапе не создаётся.
