# Дизайн: перевод в списке, разделитель «-», инфо после задания

Дата: 2026-06-29.

## 1. Разделитель «-» и перевод в списке слов
- Между формами/частями везде использовать « - » (дефис с пробелами) вместо «—».
- `ListItem` получает поле `Translation` (`strings.Join(v.Translations, ", ")`).
- Кнопка слова в «Мои слова»/«Список слов»:
  `<иконка> to <base> - <past> - <participle> - <переводы>`,
  напр. `📘 to be - was/were - been - быть, являться`.
- Варианты формы внутри позиции — через «/» (как сейчас); синонимы перевода —
  через запятую.

## 2. Инфо после задания (верно/неверно + 3 формы + перевод)
- `correctText` снова включает перевод и использует « - »:
  `to go - went - gone - идти`.
- **Учить** (`resolveLearn`), фидбэк над следующим словом:
  - верно → `✅ Верно!\n<correctText>`;
  - неверно → `❌ Неверно. Правильно: <correctText>`;
  - «Показать» → `💡 <correctText>`.
- **Тест:**
  - верно → экран результата: `✅ Верно!\n<correctText>\n\nДобавить слово в изучение?`;
  - неверно/«Помощь» → как сейчас (с переводом и «-»).

## Файлы
- `internal/service/types.go` — `ListItem.Translation`.
- `internal/service/lists.go` — `itemForms` возвращает ещё и перевод; билдеры заполняют.
- `internal/service/check.go` — `correctText`: вернуть перевод, заменить «—» на « - ».
- `internal/service/learn.go` — `resolveLearn`: «верно»-фидбэк на успехе.
- `internal/service/test_flow.go` — на верном ответе ставить `Feedback` (инфо) для экрана результата.
- `internal/bot/screens.go` — `wordRows` (+перевод, « - », `BaseLabel`); `ScreenTestResult` показывает `Feedback` + «Добавить слово в изучение?» (без дублирования «Верно!»).

## Тесты
- `correctText` содержит `to go - went - gone - идти` (« - », перевод).
- Список: кнопка содержит перевод и « - ».
- Учить: верный ответ → `Feedback` начинается с «✅ Верно!» и содержит формы+перевод.
- Тест: верный ответ → `View.Feedback` с инфо; экран результата рендерит его.
- Обновить существующие ассерты (`lists_test`, `base_to_test`).

## Вне объёма
