# Технический дизайн: Этап 2 (статусы слов, учебный материал, интерактивные списки)

- **Дата:** 2026-06-27
- **Статус:** на согласовании
- **Дорожная карта:** [2026-06-27-irregular-verbs-bot-roadmap.md](2026-06-27-irregular-verbs-bot-roadmap.md)
- **Этап 1 (готов):** [2026-06-27-irregular-verbs-bot-stage1-design.md](2026-06-27-irregular-verbs-bot-stage1-design.md)
- **Источник требований:** [README.md](../../../README.md)

## 1. Цель этапа

Дать пользователю управлять статусами слов: набирать пул для изучения и
помечать слова. После этого этапа:

1. В главном меню — **4 кнопки** (Учебный материал · Начать обучение · Мои
   слова · Список слов).
2. **Выбор уровня** (в онбординге и при смене пула) массово помечает слова
   уровня как **🔹 к изучению**.
3. **Учебный материал** показывает по одному слова со статусом 🔹 «к изучению»;
   пользователь жмёт **✅ Готов** (→ 1️⃣ режим 1) или **❌ Скип**.
4. **Список слов** — интерактивный список по 10 слов; тап по слову меняет его
   статус (иконка); внизу ◀/▶ и **✅ Окей / ❌ Отмена** (черновик).
5. **Мои слова** — тот же список, но только слова в изучении (режим 1 / режим 2
   / выучено).

Сами квизы (Режим-1/2) — это Этап 3. Кнопка «Начать обучение» пока ведёт к
заглушке «Скоро будет».

## 2. Архитектура и принципы (без изменений)

Ports & adapters из Этапа 1. Вся логика — в `internal/service`; `bot` тонкий
(только форматирование `View`). Зависимости указывают внутрь; в `users` пишет
только `service`.

## 3. Статусы слова

`WordProgress.Status` (поле `bson:"status"`):

| Статус | Иконка | Значение |
|--------|--------|----------|
| `""` (нет записи) | — | слово из невыбранного уровня; в материале не показывается |
| `to_study` | 🔹 | «к изучению»: выбрано (через выбор уровня), ждёт разбора в материале |
| `skipped` | ❌ | пропущено |
| `mode1` | 1️⃣ | режим 1 (готов к изучению) |
| `mode2` | 2️⃣ | режим 2 |
| `learned` | ✅ | выучено |

Любой из `skipped`/`mode1`/`mode2`/`learned` пользователь может поставить
вручную в списке (квизы Этапа 3/4 — автоматический способ продвижения).

Константы в `service`: `StatusToStudy`, `StatusSkipped`, `StatusMode1`,
`StatusMode2`, `StatusLearned`.

## 4. Модель данных

В агрегат `User`:

```jsonc
{
  "settings": { "level": "elementary", "variant": "gb", "order": "alpha" },
  "state": {
    "screen": "word_list",
    "page": 0,                      // текущая страница списка (для list/my_words)
    "draft": { "take": "mode1", "be": "skipped" }  // несохранённые изменения списка
  },
  "words": {
    "take": { "status": "mode1" },
    "be":   { "status": "to_study" }
  }
}
```

- В `words` храним только слова, получившие статус (через выбор уровня или
  список). Слова невыбранных уровней в карте отсутствуют.
- `state.draft` и `state.page` используются только на экранах списков
  (`word_list`, `my_words`); вне их пусты.
- Поля для серий/кругов (`mode1_streak`, `current_round`) добавятся в Этапе 3 —
  дополнение `WordProgress`, без переделки.

## 5. Каталог слов в памяти

`service` строит индекс из загруженного списка (`service.LoadVerbs`): по `base`
и по уровню (для `alpha` — отсортировано по `base`). Материал, выбор пула и
списки читают из каталога, не из Mongo. Сидинг в Mongo (Этап 1) остаётся как
есть. Конструктор: `service.New(users UserRepository, verbs []Verb) *Service`.

## 6. Контракт `service → bot`: `View`

```go
type View struct {
    Screen   Screen
    Material *MaterialView   // экран material
    List     *ListView       // экраны word_list и my_words
    Pools    *PoolChoiceView // экран choose_pool
    Notice   string          // всплывающее сообщение (answerCallbackQuery)
}
```

- `MaterialView{ Base string; Past, Participle []string; Translations []string }`
  — формы для выбранного `variant`.
- `ListView{ Title string; Page, Pages int; HasPrev, HasNext bool; Items []ListItem }`,
  `ListItem{ Base string; Status string }` — `Status` берётся из черновика
  (если есть) или из `words`; бот рисует иконку по статусу.
- `PoolChoiceView{ AllDone bool; Options []PoolOption }`,
  `PoolOption{ Level string; Exhausted bool }`.

Все use-cases/запросы возвращают `(View, error)`. `bot.render(View)` форматирует
по `View.Screen`. Непустой `Notice` бот отдаёт через `AnswerCallbackQuery` и не
меняет экран.

## 7. Use-cases (service)

Все возвращают `(View, error)`:

| Метод | Логика |
|-------|--------|
| `OpenMenu(userID)` | Экран `main_menu` (4 кнопки). |
| `SelectPool(userID, level)` | Валидирует `level`, пишет `settings.level`; все слова уровня **без статуса** (нет записи) помечает `to_study`. Возвращает `OpenMaterial`. Вызывается из онбординга (при завершении) и из `choose_pool`. |
| `OpenMaterial(userID)` | Следующее слово со статусом `to_study` (порядок по `order`). Если таких нет → `choose_pool`. |
| `MarkReady(userID, base)` | Если `base` валиден и его статус `to_study` → `mode1`. Возвращает следующее слово материала (или `choose_pool`). |
| `MarkSkip(userID, base)` | Аналогично, `to_study` → `skipped`. Следующее слово (или `choose_pool`). |
| `OpenWordList(userID)` | Открыть `word_list`: сброс `draft={}`, `page=0`. Все 170 слов. |
| `OpenMyWords(userID)` | Открыть `my_words`: сброс `draft={}`, `page=0`. Только слова `mode1`/`mode2`/`learned`. |
| `ListPage(userID, page)` | Сменить страницу текущего списка (в границах). Рисует с учётом черновика. |
| `ToggleStatus(userID, base)` | В черновике циклически меняет статус слова: `❌→1️⃣→2️⃣→✅→❌`; слово без одного из этих статусов (нет/🔹) на первый тап → `1️⃣` (вход в цикл). Перерисовать текущую страницу. |
| `CommitList(userID)` | Применить `draft` к `words` (для `my_words` слово, ставшее не-`mode1/2/learned`, остаётся в `words` с новым статусом), очистить `draft`, вернуть `main_menu`. |
| `CancelList(userID)` | Очистить `draft`, вернуть `main_menu`. |
| `StartLearning(userID)` | Заглушка: `View{ Notice: "Скоро будет 🙂" }`. |

**Порядок (`order`):** `alpha` — по `base`; `random` — стабильная перестановка,
засеянная `userID`.

**Текущий пул и материал.** Выбор уровня лишь массово ставит `to_study`;
«Учебный материал» затем показывает **все** слова со статусом `to_study` (с
любого выбранного уровня) по одному. `settings.level` хранит последний
выбранный уровень (на отбор материала не влияет). Когда `to_study` слов не
осталось — `choose_pool` предлагает другие уровни (помечает исчерпанные «✓»);
если статус есть у всех слов всех уровней → `AllDone`.

**Черновик списка.** Тапы по словам не пишут в `words` сразу — копятся в
`state.draft`. «Окей» (`CommitList`) применяет черновик и сохраняет; «Отмена»
(`CancelList`) очищает черновик. Пагинация рисует иконки с учётом черновика.

## 8. Экраны и callback-данные

Экраны (`Screen`): `main_menu`, `material`, `choose_pool`, `word_list`,
`my_words`.

| Экран | Текст / кнопки |
|-------|----------------|
| `main_menu` | «Главное меню:» · `[📖 Учебный материал]=menu:material` · `[🎓 Начать обучение]=menu:learn` · `[📋 Мои слова]=menu:my_words` · `[📚 Список слов]=menu:list` |
| `material` | `🔤 BASE (past, participle)` + `📝 переводы` · `[✅ Готов]=ready:<base>` `[❌ Скип]=skip:<base>` · `[⬅️ Меню]=nav:menu` |
| `choose_pool` | «Слова к изучению закончились. Выберите уровень:» · кнопка на каждый уровень `[<Label> (✓ если исчерпан)]=pool:<level>` · `[⬅️ Меню]=nav:menu`. Если у всех слов всех уровней есть статус (`AllDone`) — «Вы разобрали все слова.» · только `[⬅️ Меню]` |
| `word_list` / `my_words` | Заголовок («📚 Список слов» / «📋 Мои слова») + «стр. P/N». **До 10 кнопок-слов**, по одной в ряд: `[<иконка> BASE]=tog:<base>`. Затем ряд навигации `[◀ Назад]=page:<P-1>` `[▶ Вперёд]=page:<P+1>` (каждая — только если страница существует). Затем ряд `[✅ Окей]=list:ok` `[❌ Отмена]=list:cancel`. Если `my_words` пуст — «Пока нет слов в изучении.» + `[⬅️ Меню]=nav:menu` |

Иконки статусов в кнопках слов: `to_study`→🔹, `skipped`→❌, `mode1`→1️⃣,
`mode2`→2️⃣, `learned`→✅, нет→(без иконки).

Карта `kind` → use-case в роутере:
`menu:material→OpenMaterial`, `menu:learn→StartLearning`,
`menu:my_words→OpenMyWords`, `menu:list→OpenWordList`,
`ready:<base>→MarkReady`, `skip:<base>→MarkSkip`,
`pool:<level>→SelectPool`, `tog:<base>→ToggleStatus`,
`page:<n>→ListPage`, `list:ok→CommitList`, `list:cancel→CancelList`,
`nav:menu→OpenMenu`.

`tog`/`page`/`list:ok`/`list:cancel` действуют на текущий список — `word_list`
или `my_words` — по `state.screen`.

## 9. Онбординг (изменение Этапа 1)

Завершение онбординга (после выбора порядка) дополнительно вызывает «выбор
пула» для выбранного уровня: все его слова без статуса → `to_study`. Так у
нового пользователя сразу есть очередь в «Учебном материале».

## 10. Обработка ошибок

- Невалидный `base`/`level`, слово не в нужном состоянии (напр. `ready` для
  слова не `to_study`), неизвестный `kind`, страница вне границ → тихо
  подтвердить callback (как в Этапе 1), экран не ломается.
- Ошибки Mongo логируются; состояние FSM (включая черновик) в БД сохраняется.

## 11. Тестирование (TDD — тесты до кода)

`service` (фейковый `UserRepository` + реальный каталог):
- `SelectPool` ставит `to_study` всем словам уровня без статуса и не трогает уже
  решённые; невалидный уровень не меняет состояние.
- `OpenMaterial` отдаёт `to_study`-слова по `order`; когда их нет → `choose_pool`
  (исчерпанные уровни помечены; все слова со статусом → `AllDone`).
- `MarkReady`/`MarkSkip`: `to_study`→`mode1`/`skipped`, далее следующее слово.
- `ToggleStatus`: цикл `❌→1️⃣→2️⃣→✅→❌`; первый тап с «нет/🔹» → `1️⃣`; пишет в
  `draft`, не в `words`.
- `ListPage` границы; `OpenWordList`/`OpenMyWords` сбрасывают черновик/страницу;
  фильтр `my_words` = `mode1/mode2/learned`.
- `CommitList` применяет черновик к `words`; `CancelList` отбрасывает.
- `StartLearning` → `Notice`, экран не меняется.

`bot` — `render(View)` для новых экранов (иконки статусов, пагинация, кнопки
Окей/Отмена, пустое «Мои слова»).

## 12. Definition of Done

- Unit-тесты проходят; `go vet` чист.
- Ручной тест в Telegram: онбординг → в «Учебном материале» есть слова уровня →
  Готов/Скип разбирают очередь → по исчерпании предлагается другой уровень →
  «Список слов» по 10, тап меняет иконку, Окей сохраняет / Отмена сбрасывает →
  «Мои слова» показывает только слова в изучении.

## 13. Вне Этапа 2 (следующие этапы)

- Квизы Режим-1 (Этап 3) и Режим-2 (Этап 4): учат **только** слова, доведённые
  до `mode1` (т.е. показанные/решённые пользователем), и автоматически
  продвигают статус `mode1`→`mode2`→`learned`. Серии, круги, дистракторы.
- Напоминания и переход на следующий уровень (Этап 5).
- Дружелюбное сообщение при ошибке Mongo (как в Этапе 1 — отложено).

## 14. Ключевые решения (на проверку)

- Цикл статуса по тапу: `❌→1️⃣→2️⃣→✅→❌`; слово без статуса (нет/🔹) по первому
  тапу → `1️⃣`.
- Повторный выбор уровня не сбрасывает уже решённые слова (только «нет» →
  `to_study`).
- «Мои слова» — редактируемый список (тот же компонент, что «Список слов»),
  отфильтрованный по `mode1/mode2/learned`.
- «Учебный материал» показывает `to_study`-слова со всех выбранных уровней (не
  ограничен текущим уровнем).
