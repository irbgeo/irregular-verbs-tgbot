# Lists v3 (back emoji + three forms) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** List word buttons show all three forms (`BASE — past — participle` for the user's variant), and the control row's back button uses ↩️ (not 🔙).

**Architecture:** `ListItem` gains `Past`/`Participle` strings filled by the service builders from the in-memory catalog per `u.Settings.Variant`; the bot renders the three-form label and swaps the back emoji.

**Tech Stack:** Go 1.26, MongoDB, go-tgbot.

## Global Constraints

- Module `github.com/irbgeo/irregular-verbs-tgbot`; Go 1.26; deps go-tgbot + mongo-driver.
- Dependency rule: `service` imports no adapter; `bot` imports `service`.
- Word button = `<status-icon> BASE — past — participle`, forms of the chosen `variant`, multiple forms joined by `/`. No translation shown.
- Control row order/show unchanged; only back emoji 🔙 → **↩️**. Row: ↩️ `list:back` (always) · ⬅️ `lp:` (HasPrev) · ❌ `list:cancel` + ✅ `list:ok` (Dirty) · ➡️ `lp:` (HasNext).
- Reference spec §17: `docs/superpowers/specs/2026-06-28-irregular-verbs-bot-lists-design.md`.

func (s *Service) itemForms(u *User, base string) (past, participle string) {
	v, ok := s.verb(base)
	if !ok {
		return "", ""
	}
	variant := u.Settings.Variant
	return strings.Join(v.Past[variant], "/"), strings.Join(v.Participle[variant], "/")
}
